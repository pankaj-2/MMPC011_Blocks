## Chapter 1: Concept of Organizational Behaviour

> OB studies how people behave at work and how organisations behave in their environment. It covers both individual (Micro OB) and organisational (Macro OB) levels.

---

### 1.1 What is OB?

- **Micro OB** = behaviour of people *inside* organisations (attitudes, motivation, personality)
- **Macro OB** = behaviour of organisations *within* their social, political, and economic environment
- OB draws from psychology, sociology, anthropology, economics, and political science
- OB = study of human behaviour in organisations to improve performance and achieve goals

**Definition (3 key features):**
```
1. Studies human behaviour
2. In the context of organisations
3. To improve organisational effectiveness
```

---

### 1.2 Scope of OB (3 Focus Areas)

```
┌─────────────────────────────────────────────────┐
│  INDIVIDUAL          GROUP           ORGANISATION│
│  (personality,       (norms, roles,  (structure, │
│  attitude,           conflict,       culture,    │
│  perception)         leadership)     systems)    │
└─────────────────────────────────────────────────┘
```

- **Individual focus** → explores personal differences in motivation, learning, perception
- **Group focus** → studies team norms, conflict, roles, leadership
- **Organisation focus** → looks at structure, policy, and culture

---

### 1.3 Nature of OB

- OB is interdisciplinary (combines many social sciences)
- Analyses behaviour at 3 levels: individual, group, organisation
- Is both a science (systematic study) and an art (applies knowledge)
- Is action-oriented and goal-directed
- Aims to fulfil both individual AND organisational goals

---

### 1.4 Open System & Institutional Perspectives

**Open Systems View (Katz & Kahn, 1966):**

```
ENVIRONMENT
    ↓ inputs (energy, resources)
[ORGANISATION] — transforms → outputs (products/services)
    ↑ feedback loop
```

Key characteristics of an open system:
- Takes inputs from environment → transforms → exports output
- Runs in a cycle (output funds next cycle)
- Builds negative entropy (imports more energy than it uses — stays alive)
- Has feedback mechanisms to self-correct
- Moves toward specialisation over time
- Can reach same goal via different paths (equifinality)

**Institutionalisation** = creating norms to regulate behaviour of organisations and society
- Sets rules for resource allocation and sanctions for breaking norms
- Old institutionalists: society imposes norms on organisations
- Neo-institutionalists (Meyer & Scott): norms can be fictional/mythical; organisations should participate in creating them

**Neo-Institutionalist 4 Ways to Integrate Units:**
1. **Representational rules** — shared logic, taken-for-granted reality
2. **Consultative rules** — create identities linked to behaviours
3. **Normative rules** — expectations reinforced by others' actions
4. **Enforcement mechanisms** — surveillance + sanctions for compliance/deviance

**Institutional Change Process (Greenwood et al., 2002):**

```
Destabilisation → Deinstitutionalisation → Pre-institutionalisation
       ↓
Theorization → Diffusion → Full Institutionalisation
```

| Stage | What Happens |
|---|---|
| Destabilisation | Crisis/jolt disrupts existing practice |
| Deinstitutionalisation | New players disturb old consensus |
| Pre-institutionalisation | Local innovations emerge |
| Theorization | Abstract categories developed |
| Diffusion | Innovations spread for legitimacy |
| Full Institutionalisation | Ideas taken for granted |

**Isomorphism** = organisations copy each other's structures
- **Coercive** — copying due to power/compliance pressure
- **Mimetic** — copying to reduce uncertainty
- **Normative** — copying linked to professionalism

### 1.5 Society–Environment–Organisation Interface

**Sociotechnical Theory (Tavistock Group):**
- Enterprise = open system in continuous exchange with environment
- Must maintain steady state despite environmental change

**4 Environment Types (Eric Tryst & Fred Emery):**
```
Placid Random → Placid Clustered → Disturbed Reactive → Turbulent
(most stable)                                           (most chaotic)
```

**Thompson's 4 Environment Categories + Responses:**

| Environment Type | Organisation Response |
|---|---|
| Homogeneous-stable | Standard rules, few divisions |
| Heterogeneous-stable | Many divisions, rules-heavy |
| Homogeneous-variable | Geographically decentralised |
| Heterogeneous-variable | Differentiated + decentralised |

**Thompson's 4 Methods to Protect Technical Core:**
1. Buffering — absorb shocks before they hit the core
2. Smoothing — level out input/output fluctuations
3. Forecasting — anticipate changes
4. Rationing — control access when demand exceeds supply

**Burns & Stalker:**
- **Mechanistic** systems → suited for stable environments
- **Organic** systems → suited for changing, uncertain environments

**Lawrence & Lorsch (Contingency Theory):**
- **Differentiation** = breaking the organisation into subsystems to match environment
- **Integration** = collaboration between departments to achieve unified effort

### 1.6 Social Processes as Sources of Human Behaviour

Human behaviour is **learnt** — shaped through socialisation, acculturation, and assimilation.

**Social Processes (McIver and Page):**

```
Contact → Adjustment/Accommodation → Conflict
    ↓
Competition → Cooperation → Assimilation → Acculturation → Socialisation
```

| Process | Key Point |
|---|---|
| **Contact** | 3 types: physical (proximity), social (mutuality), psychological (propinquity = deep connection) |
| **Adjustment-Accommodation** | Accept unfavourable conditions to keep the system going |
| **Conflict** | Arises when goals clash; can be intra-personal, interpersonal, or organisational |
| **Competition** | Struggle for scarce resources; needs fair rules or it becomes conflict |
| **Cooperation** | Positive process; promotes affiliation and coalition |
| **Assimilation** | Taking in and absorbing new ideas/realities |
| **Acculturation & Socialisation** | Perpetuate and renew society; enable long-term change |

**Conflict Handling Strategies:**
```
Win-Lose → one party gains, other loses
Lose-Lose → both parties lose (most dangerous)
Win-Win → ideal but rarely fully achieved
```

### 1.7 Process Levels: Micro to Macro (Pareek, 2007)

Nine levels operate simultaneously in any organisation:

| Level | Process Type | Key Idea |
|---|---|---|
| Person | Existential | Self-awareness for personal growth |
| Inter-person | Empathic | Build mutual relationships |
| Role | Coping | Handle role ambiguity and conflicts |
| Group | Building | Form cohesive teams with clear norms |
| Intergroup | Collaborative | Promote cooperation between groups |
| Organisation | Growth | Continuous self-renewal and learning |
| Org-Environment Interface | Influence | Proactively respond to external changes |
| Community | Social Awareness | CSR; mutual awareness between org and community |
| Society | Value | Align societal, organisational, individual values |

| Period | Key Feature |
|---|---|
| Pre-Muslim | Intellectual/spiritual focus; fragmented kingdoms |
| Muslim Period | Invasions; forced spread of Islam |
| Initial British Colonial | Colonialism; English-educated elite; social reforms |

## Chapter 2: Approaches to Organizational Behaviour

> OB is the knowledge and skills system that helps managers understand, predict, and control human behaviour at work. This chapter covers OB's components, systems model, and four key management models.

---

### 2.1 Meaning and Approaches to OB

- OB = body of knowledge + people skills to **understand, analyse, predict, and control** human behaviour (Fred Luthans)
- Uses **interdisciplinary approach**: psychology, sociology, anthropology, economics, philosophy, ethics, law
- Also uses:
  - **Human Resources approach** → develops individuals, groups, organisations
  - **Contingency approach** → behaviour depends on situation (actors, time, place)
  - **Systems approach** → combines all approaches; most widely used

---

### 2.2 OB Components: Organisation + Behaviour

**Organisation = People + Structure + Technology**

```
         Technology
            /\
           /  \
          /    \
  People ——————— Structure
```

- **People**: Individual differences, whole person (Gestalt), human dignity
- **Technology**: Tools, methods, skills; determines structure and job roles
- **Structure**: Authority hierarchy + functional specialisation

**Key Principles of Organisational Structure (Bureaucracy — Max Weber):**
- Chain of command → direction flows top to bottom
- Span of control → ratio of subordinates to one manager
- Authority is institutional, not personal
- Authority can be delegated; responsibility cannot
- Accountability is bottom-up

**Dysfunctions of Bureaucracy:**
- Specialisation creates conflict (line-staff conflict)
- Hierarchy blocks upward communication
- No real "ideal bureaucrat" — impersonality is impossible
- Rules become more important than goals ("red tape")

**Modern Organisation Designs:**
- **Project Design** — time-bound goal; cross-functional; breaks bureaucracy
- **Matrix Design** — project structure superimposed on functional structure; laterally spread
- **Modular/Network Design** — central hub + outsourced units (used by MNCs)

---

### 2.3 Approaches to the Study of Behaviour

```
S → R         (Stimulus-Response; Classical Conditioning — Pavlov)
R → S         (Response-Stimulus; Operant Conditioning — Skinner)
S → O → R     (Stimulus-Organism-Response; individual differences matter)
A → B → C     (Antecedents-Behaviour-Consequences; converges S-R and R-S)
S → O → B → C (SOBC Model — Fred Luthans; most comprehensive)
```

**Key Models:**

| Model | Theorist | Core Idea |
|---|---|---|
| Psychoanalytic (Id-Ego-Superego) | Freud | Inner conflict drives behaviour |
| Cognitive | Edward Tolman | Behaviour is purposive and free-willed |
| Classical Conditioning (S-R) | Pavlov | Behaviour caused by stimulus |
| Operant Conditioning (R-S) | Skinner | Behaviour shaped by consequences |
| Social Learning (SOBC) | Bandura + Luthans | Person, behaviour, environment interact continuously |

**Social Learning Theory (Bandura):**
```
     Person
    ↗      ↘
Behaviour ↔ Environment
```
All three factors continuously influence each other.

---

### 2.4 Systems View of OB

```
INPUT                 THROUGHPUT                    OUTPUT
(Objectives   →  [FOS] ↔ [IS] ↔ [SS]  →   Organisational
 & Resources)         ↓   OB   ↑               Effectiveness
                  Feedback Loop
```

- **FOS** = Formal Organisation System (structure, technology, authority)
- **IS** = Individual System (personal values, needs, skills)
- **SS** = Social System (group norms, informal networks)
- FOS drives the system; IS and SS respond to it

**Formal Organisation System (FOS) Flow:**

```
Objectives & Resources → Technology → Structure → Formal Requirements
→ Job Satisfaction → Tendency to Perform
         ↑
      Authority
```

---

### 2.5 Four OB Models (Keith Davis)

```
AUTOCRATIC → CUSTODIAL → SUPPORTIVE → COLLEGIAL
(power)       (money)      (leadership)  (teamwork)
```

| Variable | Autocratic | Custodial | Supportive | Collegial |
|---|---|---|---|---|
| Depends on | Power | Economic Resources | Leadership | Mutual Contribution |
| Manager's focus | Authority | Material Rewards | Support | Integration & Teamwork |
| Employee focus | Obedience | Security | Performance | Responsibility |
| Employee result | Personal dependency | Org dependency | Participation | Self-discipline |
| Needs met | Subsistence | Maintenance/Security | Higher order | Self-realisation |
| Performance | Minimum | Passive cooperation | Awakened drives | Enthusiasm |
| Morale measure | Compliance | Satisfaction | Motivation | Commitment |

**When to use which model:**
- **Autocratic** → basic, unskilled workers; physiological need level
- **Custodial** → when economic security matters most
- **Supportive** → educated, self-motivated employees; complex technology
- **Collegial** → intellectual, flexible, project-based work; professionals
